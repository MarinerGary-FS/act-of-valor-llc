# Components — Act of Valor

## Core Objective
Each component must:
- Reduce user hesitation
- Guide immediate action
- Reinforce trust and clarity

Components are not decorative.
They are functional conversion tools.

---

## 1. HERO SECTION (PRIMARY CONVERSION ENTRY)

### Structure:
- Headline (clear, reassuring)
- Subtext (services + same-day response)
- Trust line (law enforcement + community trust)
- Primary CTA (Call Now — Immediate Help)
- Secondary CTA (Request Same-Day Service)

### Behavior:
- CTA must be visible without scrolling
- Phone CTA must be dominant
- Mobile: CTA must be thumb-accessible

### Visual Rules:
- Clean layout
- Subtle background texture (water/smoke)
- No clutter

---

## 2. SERVICE CARDS (DECISION ENGINE)

### Structure (REQUIRED per card):
- Problem-based headline
- Immediate action instruction (what user should do now)
- “What we do” (3–4 bullets max)
- CTA

### Services (in order):
1. Mold Remediation
2. Water Damage
3. Trauma / Biohazard
4. Fire Damage

### Behavior:
- Hover: slight lift + shadow
- Click: leads to dedicated section or page

### Goal:
Help user identify their situation in under 5 seconds

---

## 3. CREDENTIALS SECTION (HIGH-TRUST MODULE)

### Structure:
- Section headline
- Grid layout of certifications
- Each item includes:
  - Credential name
  - Issuing organization
  - Logo/image
  - Alt text

### Behavior:
- Hover: slight emphasis or highlight
- Optional tooltip or label

### Rules:
- Must feel premium and authoritative
- Must support adding new credentials easily

---

## 4. TRUST SECTION (EMOTIONAL + AUTHORITY LAYER)

### Title:
Why Atlanta Trusts Act of Valor

### Structure:
- Short emotional statement
- 3–5 trust bullets:
  - Compassion
  - Law enforcement relationships
  - Same-day response
  - Professionalism

### Behavior:
- Static (no heavy animation)
- Easy to scan

---

## 5. RESOURCES SECTION (SEO AUTHORITY MODULE)

### Structure:
Categorized links:

- Weather & Storm Tracking
- Atlanta / Georgia Public Safety
- Emergency Preparedness
- Insurance & Claims
- Restoration Education

### Behavior:
- Clean list or card layout
- Links open in new tab

### Rules:
- Must support SEO authority
- No irrelevant links

---

## 6. PUBLIC SAFETY UPDATES (LOCAL RELEVANCE MODULE)

### Title:
Atlanta Public Safety Updates

### Structure:
- Feed-style layout (3–5 items)
- Title + short description + link

### Behavior:
- Scrollable or static list
- Must not overwhelm page

### Rules:
- Focus on Atlanta / Georgia relevance
- Must feel credible and current

---

## 7. CTA BLOCKS (CONVERSION REINFORCEMENT)

### Structure:
- Short urgency-driven text
- Primary CTA (Call Now)
- Secondary CTA (Request Service)

### Placement:
- After services
- After trust section
- Before footer

### Behavior:
- Visually distinct from other sections
- High contrast

---

## 8. STICKY HEADER (PERSISTENT ACTION BAR)

### Structure:
- Logo (left)
- Navigation (center/right)
- Phone number (clickable)
- CTA button

### Behavior:
- Always visible on scroll
- Shrinks slightly when scrolling
- Mobile: simplified layout

---

## 9. FOOTER (FINAL CONVERSION + NAVIGATION)

### Structure:
- Phone number (clickable)
- Service list
- Locations served
- Social media links
- Quick navigation
- Final CTA

### Behavior:
- Clean layout
- No clutter
- All links functional

---

## 10. CONTACT / FORM COMPONENT

### Structure:
- Name
- Phone
- What happened
- Location

### Behavior:
- Instant submission feedback
- Mobile optimized
- Minimal typing required

### Rules:
- Must be fast and frictionless
- No unnecessary fields

---

## COMPONENT PRIORITY ORDER

1. Hero
2. Sticky Header (Phone CTA)
3. Service Cards
4. CTA Blocks
5. Trust Section
6. Credentials Section
7. Contact/Form
8. Resources
9. Public Safety Updates
10. Footer

---

## FINAL RULE

Every component must answer:
"What should the user do next?"

If it does not guide action, it should not exist.
